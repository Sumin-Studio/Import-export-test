## Rory’s evolution story for the agent (saved for later)

*Rory’s follow-up thinking on how the Safety Shield story could evolve.*

### Beta (pre–Xerocon?)

- Safety Shield is **scanning all bills in the background** (for whatever scenarios are agreed for beta).
- **Surfacing risk** in Bills List view + bill detail screens.
- **Outcome:** User gets a set of **recommended actions** they can take to resolve risky bills.

### GA — Xerocon + 90 days

- Safety Shield is **plugged into bill approval agent + cashflow agent** as an input into decision making.
- Safety Shield (or maybe JAX) can **be engaged to fix the issue** for the user — e.g. spin up an email to the vendor to query the bill, or do a COP check to test a new supplier bank account.

### Long-term future

- Safety Shield **operating across other parts of Xero** (e.g. payroll).
- User could **configure rules** for certain types of bills or vendors on when to flag them, or the system could **learn behaviours** of users over time.
- **Synergy:** If we give customers the ability to set rules for a bill approval agent, there could be nice overlap with configurable Safety Shield rules.

*All just thoughts for now.*
