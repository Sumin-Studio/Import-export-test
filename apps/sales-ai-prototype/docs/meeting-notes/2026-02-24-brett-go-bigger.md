## The Breakthrough

Brett's core challenge: **"It's an agent. It can be zero taps. Because we're doing everything for you. And that's the ah-ha moment."**

### The Evolution
1. Started with chatbots in a JAX drawer
2. Reduced to 3 clicks
3. Moved to dashboard widgets, WhatsApp, ChatGPT
4. **Brett: still not ambitious enough. Diya will not be moved.**
5. Zero taps. It's just done. Don't worry.

## The Super Bowl Commercial Approach

Start with the **top of the pyramid** (ideal outcome), then explain the foundational steps that enable it.

### Microsoft Bing Parallel
One slide. Three images:
- Person at concert → **Bing Audio** (like Shazam)
- Person photographing a French menu → **Bing Vision**
- Person doing a search → **Bing Search**

That was the pitch. Instantly understood: Bing should be multi-modal. Detail slides follow.

### Jon's Version for Diya
Three screens:
1. **Phone alert** — "Bills have been paid. Click to learn more."
2. **WhatsApp alert** — "Items have been auto-approved. Click to learn more."
3. **ChatGPT conversation** — Someone pays someone without creating a bill.

Then come prepared with how you train the system, how trust is built, etc.

## Brett's Key Principles

1. **Lead with the ideal experience** where bill payment is dissolved into the background
2. **Then explain how trust is built** — users start wanting control, eventually want less "human in the loop"
3. **Don't present an already-compromised solution** — better to be asked to scale back ambition than be criticized for lack of it
4. **Go further: eliminate Xero login entirely** — user gets a text about paid bills, contractor commands payment via voice
5. **The AI is strategic, not dumb** — uses multiple data inputs: funds, projected income, supplier relationships, non-Xero data (emails, phone calls, external accounts)
6. **Start from scratch** — imagine how the product would work if built today, focusing on seamless bill management in the background
