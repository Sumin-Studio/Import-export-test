# Bill AI Automation — Docs

**Convention:** All documents use date-prefixed filenames `yyyy-mm-dd-(description).md` where the date is when the doc was created or the event occurred. Living docs in `status/` (PROJECT-STATUS, INDEX) are not date-prefixed.

**Subfolders by type:**

| Folder | Contents |
|--------|----------|
| **status/** | Single source of truth: PROJECT-STATUS.md (latest state, this week, open actions), INDEX-whats-stored-and-next-steps.md (inventory). Start here. |
| **ai-daily-journal/** | One file per day (yyyy-mm-dd.md) with a detailed summary of what was discussed that day. |
| **meeting-notes/** | Captures from meetings, Slack threads, and planning sessions. Filenames: yyyy-mm-dd-(topic).md |
| **workshop/** | Workshop run sheets, prep, problem lists, scored rationale, and facilitation guides. Filenames: yyyy-mm-dd-(description).md |
| **strategy/** | Strategic context, principal brief, POC scope, competitive analysis. Filenames: yyyy-mm-dd-(description).md |
| **reference/** | Research links, people briefs, how-to (e.g. Google Sheets automation). Filenames: yyyy-mm-dd-(description).md |
| **raw-data/** | Unprocessed or semi-processed output (e.g. synthetic user quotes). Re-run processes documented in reference/. |

**Start here:** [status/PROJECT-STATUS.md](status/PROJECT-STATUS.md) for latest state. [status/INDEX-whats-stored-and-next-steps.md](status/INDEX-whats-stored-and-next-steps.md) for full inventory. For recent repo changes, see [CHANGELOG.md](../CHANGELOG.md) at repo root.
