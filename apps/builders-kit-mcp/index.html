Builders Kit

builders-kit-mcp

A minimal Model Context Protocol server, deployed as Vercel serverless functions. Point an MCP client at /mcp and start calling tools.

Endpoints

POST /mcp — MCP Streamable HTTP transport
GET /health — liveness check

Connect from Cursor (bring your own tokens)

The server stores no credentials. Every caller passes their own tokens as HTTP headers. Edit ~/.cursor/mcp.json and add the headers for the tools you want to use — omit the ones you don't. 

{
  "mcpServers": {
    "builders-kit-mcp": {
      "type": "http",
      "url": "https://builders-kit-mcp-xro.vercel.app/mcp",
      "headers": {
        "X-Figma-Token": "figd_...",
        "X-OpenAI-Api-Key": "sk-...",
        "X-Confluence-Base-Url": "https://yoursite.atlassian.net",
        "X-Confluence-Email": "you@xero.com",
        "X-Confluence-Token": "ATATT..."
      }
    }
  }
}

After editing, toggle the builders-kit-mcp server off/on in Cursor Settings → MCP so it re-reads the config.

Where to get each token

  • Figma — figma.com/settings → Security → Personal access tokens → Generate new token. Read-only scopes are enough. The token inherits all your Figma permissions, so treat it like a password.
  • OpenAI — platform.openai.com/api-keys.
  • Confluence — id.atlassian.com/manage-profile/security/api-tokens. Base URL is your Atlassian site root (e.g. https://xero.atlassian.net).

Tools

  • hello_world — friendly greeting. Verifies the server is reachable. No token needed.
  • figma_get_tokens, figma_get_components, figma_get_file_structure, figma_export_images, figma_setup_guide — read-only Figma queries + signed image export URLs. Needs X-Figma-Token.
  • confluence_search, confluence_get_page, confluence_list_spaces — needs the three X-Confluence-* headers.
  • transcribe_url — Whisper transcription from a public URL. Needs X-OpenAI-Api-Key.
  • userdata_query, userdata_get_insights — stubbed; return example shapes until wired to a real source.

Why headers, not server-side env vars?

This endpoint is reachable on the public internet, so baking a token into the server would effectively publish it. Per-user headers keep credentials on each person's laptop and make every Figma / Confluence / OpenAI call traceable back to the user who made it. Env vars still work as a fall-back for local dev.
